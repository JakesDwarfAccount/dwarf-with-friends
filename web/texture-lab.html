    1. 01

      Bank the mismatch

      Capture the browser and the native/Steam reference for the same subject, state, material, camera, and renderer. Record the exact identity—not just “that red thing.”

    2. 02

      Trace, do not guess

      Use this lab to find the wire field, map key, fallback branch, source sheet, and current cell. Decide whether the information was lost, never generated, resolved incorrectly, or drawn incorrectly.

    3. 03

      Generate an adjudication picker

      python tools/spritepick/gen_picker.py --token "<exact subject>"

      The picker shows current pixels beside banked references and candidates. A human choice is the authority for ambiguous art.

    4. 04

      Patch the authoritative source

      python tools/spritepick/apply_choices.py <choices.json>

      Prefer the generator or resolver rule that owns the class. Record deliberate exceptions in tools/spritepick/pins.json so regeneration cannot silently revert them.

    5. 05

      Test both absence and wrong-art

      python tools/spritepick/verify_pins.py
python tools/harness/texture_coverage_audit.py --selftest

      A “non-empty sprite” check is insufficient. The audit’s seeded-bad cases must prove it can reject an existing-but-wrong cell.

    6. 06

      Re-run the independent oracle

      Verify the actual subject again in both renderers when both paths implement the class. After a web copy, hard-refresh; an already-open tab can keep old JavaScript through server restarts.

    Four invariants worth memorizing

    • Missing and wrong are different test classes. A fallback can turn missing data into confident wrong art.
    • Maps are derived artifacts. Fix their generator or pin an adjudicated exception; do not rely on a hand edit that regeneration erases.
    • Material and silhouette are separate decisions. The right pick shape with the wrong palette row is still wrong; the right color on an item box is still wrong.
    • The native window is the visual oracle. /frame.jpg and /tiledump miss downstream see-down fog, and passive viewport texpos is native-camera-only.
